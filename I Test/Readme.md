Test Plan: Login Form & ReqRes API (Trainee QA)
1. Scope (Область тестирования)

    Web: Авторизация на странице https://herokuapp.com (Desktop-версия).
    Mobile Web: Тестирование адаптивности и мобильной специфики той же формы через мобильный браузер.
    API: Проверка базовых CRUD-операций (GET, POST, PUT, DELETE) сервиса https://reqres.in.

2. Types of Testing (Виды тестирования)

    Functional: Проверка работы логики входа и корректности ответов API.
    Negative: Тестирование обработки ошибок (неверные данные, пустые поля, некорректные ID в API).
    UI/UX: Проверка верстки, доступности элементов и читаемости сообщений об ошибках.
    Mobile Testing: Проверка прерываний (звонки), смены ориентации экрана и работы с экранной клавиатурой.
    API Testing: Валидация статус-кодов и структуры JSON-ответов.

3. Environments & Tools (Среда и инструменты)

    Browsers: Google Chrome (Desktop), Safari/Chrome (Mobile).
    Devices: Реальный смартфон (Android).
    API Tool: Postman (Desktop version).
    Documentation: GitHub.

4. Risks & Limitations (Риски и ограничения)

    Данные: Используются публичные песочницы (The Internet, ReqRes), данные в которых могут быть сброшены или изменены третьими лицами.
    Доступ: Отсутствие доступа к БД и логам сервера, что ограничивает глубину анализа причин багов.
    Время: Ограничение в 3–5 часов не позволяет провести полное регрессионное тестирование и нагрузочные тесты.
    Специфика: Тестируется только мобильный браузер, а не нативное приложение (Native App).
